    Covariate Balance¶

    Overview¶

    In observational studies, treatments are not randomly assigned—making it critical to assess whether treatment groups have similar distributions of baseline (pre-treatment) covariates. Covariate balance measures whether the treatment and control groups look similar on observed characteristics. Poor balance can introduce bias in causal estimates.

    samesame provides classifier two-sample tests (CTSTs) as a flexible tool for assessing balance. The key idea: if a classifier can reliably distinguish treatment from control based on covariates alone, the groups are imbalanced.

    When to use this¶

    • After matching or weighting: Verify that your matching or inverse probability weighting (IPW) strategy actually balanced the groups.
    • Sensitivity analysis: Test balance across many covariate combinations without specifying a single summary metric.
    • Production monitoring: Check that covariate distributions remain stable over time in observational systems.

    Example: Testing balance after matching¶

    Below is a practical example using the lalonde dataset from the MatchIt R package, which compares an experimental job training program (treatment) to a control group on baseline covariates like age, education, race, marital status, and prior earnings.

    Load data and compute propensity scores¶

    import pandas as pd
import numpy as np
from sklearn.linear_model import LogisticRegression

# Load lalonde dataset from MatchIt R package
url = "https://raw.githubusercontent.com/kosukeimai/MatchIt/master/data/lalonde.tab"

# Read the first line to extract column headers, then read the rest as data
with pd.io.common.urlopen(url) as response:
    # Read first line as column headers
    header_line = response.readline().decode('utf-8').strip().split('\t')
    # Read remaining lines as data
    lalonde = pd.read_table(response, sep='\t', header=None, names=header_line)# Display basic info
print(f"Dataset shape: {lalonde.shape}")
print(lalonde.head())

# Covariates and treatment
covariates = ['age', 'educ', 'race', 'married', 'nodegree', 're74', 're75']
X = lalonde[covariates].copy()

# Convert categorical (race) to dummies
X = pd.get_dummies(X, columns=['race'], drop_first=False)
X = X.astype(float)

treatment = lalonde['treat'].values

    Before matching: testing imbalance¶

    from samesame.ctst import CTST
from sklearn.metrics import roc_auc_score

# Fit a simple logistic regression classifier on raw (unmatched) data
clf = LogisticRegression(random_state=42, max_iter=1000)
clf.fit(X, treatment)
y_pred_proba = clf.predict_proba(X)[:, 1]

# Test for imbalance using CTST
ctst_before = CTST(actual=treatment, predicted=y_pred_proba, metric=roc_auc_score)
print(f"Before matching:")
print(f"  AUC: {ctst_before.statistic:.3f}")
print(f"  p-value: {ctst_before.pvalue:.4f}")

    Example output:

    Before matching:
  AUC: 0.711
  p-value: 0.0000

    High AUC (> 0.5) and low p-value indicate the groups are imbalanced—the classifier can reliably predict treatment assignment from covariates alone.

    After matching: testing balance¶

    Perform 1:1 nearest-neighbor matching on propensity scores:

    from sklearn.neighbors import NearestNeighbors

# Compute propensity scores
propensity_model = LogisticRegression(random_state=42, max_iter=1000)
propensity_model.fit(X, treatment)
propensity_scores = propensity_model.predict_proba(X)[:, 1]

# 1:1 nearest-neighbor matching on propensity scores (matching treated to control)
treatment_idx = np.where(treatment == 1)[0]
control_idx = np.where(treatment == 0)[0]

# Find nearest neighbors in control group for each treated unit
nbrs = NearestNeighbors(n_neighbors=1, algorithm='ball_tree').fit(
    propensity_scores[control_idx].reshape(-1, 1)
)
distances, indices = nbrs.kneighbors(propensity_scores[treatment_idx].reshape(-1, 1))

# Create matched dataset
matched_treatment_idx = treatment_idx
matched_control_idx = control_idx[indices.ravel()]
matched_idx = np.concatenate([matched_treatment_idx, matched_control_idx])

X_matched = X.iloc[matched_idx].reset_index(drop=True)
treatment_matched = treatment[matched_idx]

print(f"Matched sample size: {len(matched_idx)} (treatment: {(treatment_matched == 1).sum()}, "
      f"control: {(treatment_matched == 0).sum()})")

# Test balance in matched data
clf_matched = LogisticRegression(random_state=42, max_iter=1000)
clf_matched.fit(X_matched, treatment_matched)
y_pred_proba_matched = clf_matched.predict_proba(X_matched)[:, 1]

ctst_after = CTST(actual=treatment_matched, predicted=y_pred_proba_matched, metric=roc_auc_score)
print(f"After matching:")
print(f"  AUC: {ctst_after.statistic:.3f}")
print(f"  p-value: {ctst_after.pvalue:.4f}")

    Example output:

    Matched sample size: 370 (treatment: 185, control: 185)
After matching:
  AUC: 0.503
  p-value: 0.7854

    After matching, AUC close to 0.5 and high p-value indicate the groups are now balanced—the classifier cannot reliably distinguish treatment from control using covariates alone.

    Using different classifiers and metrics¶

    Just as in the CTST examples, you can use any classifier and metric:

    from sklearn.ensemble import RandomForestClassifier
from sklearn.metrics import balanced_accuracy_score

# Use a more flexible classifier (random forest)
rf = RandomForestClassifier(n_estimators=100, random_state=42, max_depth=5)
rf.fit(X_matched, matched_treatment)
y_pred_binary = rf.predict(X_matched)

# Use balanced accuracy instead of AUC
ctst_rf = CTST(actual=matched_treatment, predicted=y_pred_binary, metric=balanced_accuracy_score)
print(f"Balanced Accuracy (Random Forest): {ctst_rf.statistic:.3f}, p-value: {ctst_rf.pvalue:.4f}")

    Practical recommendations¶

    • Choose meaningful metrics: Use AUC or balanced accuracy; these are robust to class imbalance and easy to interpret.
    • Test covariate subsets: Apply CTSTs to different covariate subsets to identify which covariates remain imbalanced.
    • Combine with domain knowledge: Statistical balance does not guarantee causal validity; use CTSTs alongside subject-matter expertise.
    • Report before and after: Show balance statistics (e.g., mean differences, AUC) alongside CTST results for transparency.
